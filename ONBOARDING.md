# Member onboarding

## New member checklist

- [ ] GitHub username verified.
- [ ] Two-factor authentication enabled.
- [ ] Code of Conduct accepted.
- [ ] Security and confidentiality rules reviewed.
- [ ] Added initially to `asr-contributors`.
- [ ] Sponsor assigned.
- [ ] First repository and task assigned.
- [ ] Development environment tested.
- [ ] Commit-signoff and pull-request workflow understood.
- [ ] 30-day access review scheduled.

## First contribution

The first contribution should be small, reviewable and useful: documentation, a test, a reproducibility check, an issue triage or a contained research task. New members must not receive broad administrator access as part of onboarding.
